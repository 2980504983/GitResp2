分支
    定义: 在版本控制中，使用多条线同时推进多个任务，这里面说的多条线就是分支
    分支就是说，开了一个项目是一个主分支，然后，要给多个人进行编码完成其功能，然后就要创建多个
    分支，这些分支互不影响，当功能创建好了后，就可以将该分支与主分支进行合并，就完成的项目的推进。
    文件夹里有图

查看分支:
    git branch -v
    (前面的星和绿色字体:表示当前所在的分支)

创建分支:
    git branch 要创建分支的名字

切换分支:
    git checkout 要去到分支的名字

将其它分支合并到当前分支:
    git merge 要合并的分支名

分支冲突:
    两个分支合并时，两个分支的同一文件，同一行不相同
    当发生分支冲突，分支不会直接合并会进入合并状态
    在合并状态下输入  （cat 冲突的文件名）  就能看到冲突的地方

    解决:
        1 在工作区中手动(点此电脑那种手动)打开冲突文件，将特殊符号删除，留下最终结果
        2 但是合并状态还没有解除，将文件重新add
        3 然后commit，注意这个commit不用带最后的文件名
